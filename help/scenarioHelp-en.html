Scenarios


In this tab, user can define scenarios.
Initial state scenario can be renamed but must not be deleted.
New scenarios can then be added with Add scenario from base scenario and landuse modifications layer. This layer must have a numeric field containing new values according to friction codes. If not, reclassification can be performed in Data tab.
Option Layer represents project extent must be set to to specify that study extent for this scenario is built from landuse modification layer. For instance this option must be checked for a linear infrastructure but not for a local mitigation.
Option Create short-term scenario duplicates current scenario in case of a mitigation which fonctionnality is not the same at short and long-term. Short-term scenario name is suffixed by Short. Ensure reclassification is well set by veryfing Data Classes table.
