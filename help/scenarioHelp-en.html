Scenarios


In this tab, user can define scenarios.
Initial state scenario can be renamed but must not be deleted.
New scenarios can then be added with Add scenario from base scenario and landuse modifications layer. This layer, vector or raster, must either contain new values according to friction codes, either be reclassified to a fixed value. Reclassification rules are displayed in Classes table of Data tab.
Option Layer represents project extent must be set to to specify that study extent for this scenario is built from landuse modification layer. For instance this option must be checked for a linear infrastructure but not for a local mitigation.
Option Create long-term scenario duplicates current scenario in case of a mitigation which fonctionnality is not the same at short and long-term. Long-term scenario name is suffixed by Long. Ensure reclassification is well set by veryfing Data Classes<.i> table.
